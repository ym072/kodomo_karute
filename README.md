# サービス概要
子どもの体調不良を「1回の体調不良単位」で管理できる病状記録アプリです。

看病中の保護者が最小限の操作で症状を記録し、現在の状態や経過をすぐに把握できることを目的に開発しました。

発熱の推移や下痢・嘔吐の回数を手軽に記録でき、体調不良の開始から完治までを1つのまとまりとして管理します。

記録内容はサマリー表示され、過去の履歴も簡単に確認できます。



# 開発の背景
息子が胃腸炎にかかり、昼夜を問わず1日に30回ほど下痢を繰り返したことがありました。

看病をする中で、私自身も寝不足と疲労により、病院で尋ねられる「いつから」「何回」「どの程度」といった病状を正確に把握できず、うまく答えられない状況に困りました。

既存の育児アプリにも記録機能はありますが、1日ごとの表示のみで累計回数が分かりにくいなど、病状把握に特化しているとは言えず、看病中の負担を軽減するには十分ではありませんでした。

「頭がぼーっとしている状態でも、最小限の操作で記録でき、すぐに状況を把握できるサービスがあれば助かる」
そう感じたことが、本サービスを開発したきっかけです。

看病中の保護者に寄り添い、記録と確認を手軽に行える、思いやりのある病状記録サービスを目指して制作しました。



# サービスの差別化ポイント
### 類似サービス
ぴよログなどの育児記録アプリ。

症状記録機能はあるものの日ごとの表示が中心で、体調不良時には日付を遡って確認する必要があります。


### 差別化ポイント
- 体調不良単位（開始〜完治）で管理
- 症状の累計カウント機能（例：下痢回数）
- 症状と日数を自動でサマリー表示
- 過去の体調不良を一覧で確認可能

### 利用シーン例
例）「1週間前から咳鼻水、3日前から下痢、昨日発熱」の場合

→ 本サービスでは、症状の開始日・継続日数・回数をサマリーで即時確認できます。



# 機能紹介
### ユーザー登録 / 子どもの登録
- ユーザー（アプリ利用本人）登録機能
- Googleログイン機能
- パスワードリセット機能
- 子どもの複数登録機能（ユーザー1人に対して記録対象を複数登録可能）
- 画像アップロード機能（子どものアイコンのため）：看病中でもお気に入りの子どもの顔をアイコンとして表示することで、少しでも糧になるよう実装。

### 症状記録

- 症状記録機能、一部の症状についてはカウント機能含む：看病中でも素早く入力できるよう、最小限の操作で登録できるUIを意識。

### データ構造の工夫
体調不良の開始から完治までを1つの単位として管理しつつ、その中で発生する各症状を日時付きで複数記録できるようにするため、体調不良データと症状記録データを分けて設計しました。

1回の体調不良の中で数種類の症状が複数回発生することを踏まえ、体調不良を親データ、症状記録を子データとして関連付けています。

これにより体調不良全体の経過を追いながら、各症状がいつ発生したかを記録できる構造とし、そのデータを活用して一部症状の回数を算出できるようにしています。
